# 白話術語表

這份文件把 repo 常見名詞改成比較好懂的中文。

原則：

- 對話時優先用中文
- 英文只放在括號，方便對 branch、PR、檔名
- 如果某個詞還是不清楚，就把它當成「白話名稱」來看，不需要背英文

## 常見流程詞

### 主線

- 英文：`main`
- 白話：主要版本、正式開發線

### 分支

- 英文：`branch`
- 白話：一條獨立開發線
- 例子：`codex/guild-order-submit-continuation` 就是「公會訂單續做分支」

### 工作樹

- 英文：`worktree`
- 白話：同一個 repo 的另一份實體資料夾
- 用途：讓不同對話可以同時做不同任務，不會互相覆蓋

### 合併請求

- 英文：`PR`、`pull request`
- 白話：把分支內容合回主線前的審查頁

### 交接

- 英文：`handoff`
- 白話：留給下一個 Codex 或人看的接手資訊

### 自動檢查流程

- 英文：`autonomy loop`
- 白話：每次改完後固定要跑的一套檢查
- 內容通常是：`doctor`、`pytest`、`ruff`、handoff artifacts

### 品質門

- 英文：`quality gate`
- 白話：這次改動有沒有通過基本檢查

### 可交付

- 英文：`ready`
- 白話：這一輪已經可以交給下一步，不代表整個功能百分之百完成

### 小範圍修改

- 英文：`bounded change`
- 白話：只做一小塊、容易回滾的改動

## 對話與工作模式

### 派工主對話

- 英文：`dispatch thread`
- 白話：你一直下需求的主對話
- 用途：決定下一步做什麼，不直接塞很多實作細節

### 工作者對話

- 英文：`worker thread`
- 白話：專門做某一個任務的新對話

### 探索模式

- 英文：`exploration mode`
- 白話：先研究遊戲畫面和流程，不急著開 PR
- 用途：陌生 UI、路線還沒跑通時使用

### 交付模式

- 英文：`delivery mode`
- 白話：流程已經清楚，開始正式寫程式、測試、開 PR

### 檢查點

- 英文：`checkpoint`
- 白話：流程中的關鍵畫面
- 例子：主畫面、選單展開、公會訂單頁

### 截圖確認式導航

- 英文：`screenshot-gated navigation`
- 白話：先截圖確認畫面正確，再點一下，點完再截圖驗證

## 程式結構詞

### 執行層

- 英文：`runtime`
- 白話：真正把任務跑起來的核心邏輯

### 模擬器層

- 英文：`emulator`
- 白話：跟 MuMu、ADB 溝通的那層

### 視覺辨識層

- 英文：`vision`
- 白話：看畫面、找按鈕、判斷圖像狀態的那層

### 任務層

- 英文：`task`
- 白話：某個實際功能流程的規則
- 例子：公會訂單提交、每日領獎

### 真實畫面依據

- 英文：`truth contract`、`live evidence`
- 白話：有真實截圖或真機證據支持的規則，不是猜的

### 當前面板

- 英文：`current panel`
- 白話：目前畫面正在顯示的那一個區塊或頁面

## 裝置詞

### 裝置編號

- 英文：`ADB serial`
- 白話：模擬器的唯一連線代號
- 例子：`127.0.0.1:16416`

### 保留裝置

- 英文：`reserved device`
- 白話：這條對話暫時固定使用的模擬器，不和別的對話共用

## 目前常見任務中文對照

### 每日領獎

- 英文：`claim_rewards`
- 白話：領活動或日常獎勵

### 公會簽到

- 英文：`guild_check_in`
- 白話：到公會頁做簽到

### 公會訂單提交

- 英文：`guild_order_submit`
- 白話：處理公會訂單、交材料、跳過或刷新

### 商會委託

- 英文：`merchant commission`
- 白話：商會委託系統

### 喵手商團

- 英文：`merchant_commission_meow`
- 白話：商會委託裡目前第一個優先要做的商團

### 王國軍需處

- 白話：商會委託裡第二優先要做的商團

## 之後的溝通規則

之後對使用者回覆時，盡量用下面格式：

- `公會訂單提交 (guild_order_submit)`
- `商會委託 (merchant commission)`
- `探索模式`
- `交付模式`

也就是：

- 先講中文
- 英文只放括號
- 如果只是對話，不必堆一堆英文縮寫
